# Plan: Understanding the item/templatetags/ Folder

## Overview
The `item/templatetags/` folder contains custom Django template tags and filters used in the Bazzar project. This is a standard Django convention for creating reusable template functionality.

## Contents
- `__init__.py`: Marks the directory as a Python package (required for Django to recognize templatetags)
- `item_extras.py`: Contains two custom template filters:
  1. `is_new`: Determines if an item was created within a specified number of days (default 30)
  2. `has_sale`: Heuristic to check if an item appears to be on sale

## Filter Details

### is_new(value, days=30)
- Accepts either a datetime object or an object with `created_at`/`created_on`/`created` attribute
- Returns True if the item was created within the specified number of days
- Handles timezone awareness conversions safely
- Used to highlight new items in templates

### has_sale(item)
- Checks multiple attributes to determine if an item is on sale:
  1. Boolean flag `is_on_sale`
  2. Numeric `discount` > 0
  3. `sale_price` < `price` (when both exist)
- Returns False if item is None or no sale indicators found
- Used to display sale badges or special pricing in templates

## Usage in Templates
These filters would be loaded in Django templates with:
```django
{% load item_extras %}
```
Then used like:
```django
{% if item|is_new %}New!{% endif %}
{% if item|has_sale %}On Sale!{% endif %}
```

## Purpose
This folder encapsulates presentation logic that determines how items are displayed based on their properties, keeping templates clean and reusable.
